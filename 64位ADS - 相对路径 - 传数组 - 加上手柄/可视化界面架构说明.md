# 项目可视化界面架构说明

## 1. 技术栈概览

| 层次 | 技术 | 用途 |
|------|------|------|
| 主框架 | WPF (.NET 4.7.2, x64) | 应用程序主体、窗口管理、UI 布局 |

## 2. 窗口管理架构

### 2.1 核心接口 — ILocalWnd

所有可被主窗口管理的子窗口必须实现此接口：

```csharp
// 文件: plugins\plugins\Helper\Widgets\BaseWnd.cs
public interface ILocalWnd
{
    void Show();
    void Close();
    event EventHandler Closed;
}
```

### 2.2 窗口注册表 — ViewTypes

主窗口 `MainWnd.xaml.cs` 通过字典注册所有子窗口类型：

```csharp
// 文件: ErcpApp\Dialog\MainWnd.xaml.cs
private readonly Dictionary<string, Type> ViewTypes = new Dictionary<string, Type> {
    {"Input",        typeof(Dialog.InputWnd) },
    {"Inject",       typeof(Dialog.InjectWnd) },
    {"Setting",      typeof(Dialog.SettingWnd) },
    {"Daemon",       typeof(Dialog.DaemonWnd) },
    {"Module",       typeof(DecoratedWnd<NavNode.EditorView>) },
    {"Register",     typeof(DecoratedWnd<ToolApp.MainWnd>) },
    {"Recon",        typeof(DecoratedWnd<Dialog.ReconWnd>) },
    {"Navigate",     typeof(DecoratedWnd<Navigate.NavigateWnd>) },
    {"SpeedSetting", typeof(Dialog.SpeedSetWin) },
    {"ToPos",        typeof(Dialog.ToPosWndxaml) },
};
```

### 2.3 窗口实例化流程

```csharp
// 文件: ErcpApp\Dialog\MainWnd.xaml.cs
private void NormalWindow_Click(object sender, RoutedEventArgs e)
{
    if (sender is Control c && c.Tag is string tag)
    {
        if (Views[tag] == null)
        {
            var wnd = (ILocalWnd)Activator.CreateInstance(ViewTypes[tag], this);
            wnd.Closed += delegate { Views[tag] = null; };
            Views[tag] = wnd;
        }
        Views[tag].Show();
    }
}
```

流程：按钮 Tag → 查找 ViewTypes → 反射创建实例 → 缓存到 Views → 调用 Show()

## 3. 窗口基类与包装器

### 3.1 BaseWnd — 标准窗口基类

```csharp
// 文件: plugins\plugins\Helper\Widgets\BaseWnd.cs
public class BaseWnd : Window, ILocalWnd, INotifyPropertyChanged
{
    public BaseWnd(Window parent)
    {
        this.Owner = parent;
        // 设置窗口样式、事件绑定等
    }
}
```

适用场景：直接继承即可使用，适合大多数普通窗口。

### 3.2 DecoratedWnd&lt;T&gt; — 装饰器包装

```csharp
// 文件: plugins\plugins\Helper\Widgets\BaseWnd.cs
public class DecoratedWnd<T> : ILocalWnd where T : Window
{
    private T _window;

    public DecoratedWnd(Window parent)
    {
        _window = (T)Activator.CreateInstance(typeof(T), parent);
    }

    public void Show() => _window.Show();
    public void Close() => _window.Close();
    public event EventHandler Closed
    {
        add { _window.Closed += value; }
        remove { _window.Closed -= value; }
    }
}
```

适用场景：第三方窗口或不方便修改继承关系的窗口，用装饰器适配 ILocalWnd。

### 3.3 ThreadWnd&lt;T&gt; — 独立线程窗口

```csharp
// 文件: plugins\plugins\Helper\Widgets\BaseWnd.cs
public class ThreadWnd<T> : ILocalWnd where T : Window
{
    // 在独立 STA 线程中创建和运行窗口
    // 拥有独立的 Dispatcher 消息循环
}
```

适用场景：需要独立 UI 线程的重型渲染窗口，避免阻塞主线程。

## 8. 新程序集成指南

### 8.1 方案一：纯 WPF 窗口（最简单）

适用于：不涉及原生渲染的配置界面、数据展示界面。

**步骤：**

1. 在现有插件项目或新建类库项目中创建窗口类：

```csharp
// MyNewPlugin\MyVisualizationWnd.xaml.cs
public partial class MyVisualizationWnd : BaseWnd
{
    public MyVisualizationWnd(Window parent) : base(parent)
    {
        InitializeComponent();
    }
}
```

2. 在 `MainWnd.xaml.cs` 的 ViewTypes 中注册：

```csharp
{"MyVis", typeof(MyNewPlugin.MyVisualizationWnd) },
```

3. 在主窗口 XAML 中添加触发按钮（Tag 与注册名一致）：

```xml
<RibbonButton Tag="MyVis" Label="我的可视化" Click="NormalWindow_Click" />
```

4. 如果有自定义资源字典，在 App.xaml 中合并：

```xml
<ResourceDictionary Source="pack://application:,,,/MyNewPlugin;component/Theme/MyTheme.xaml" />
```

### 8.2 方案二：包装已有窗口（适配模式）

适用于：已有独立窗口，不想修改其继承关系。

```csharp
// 注册时使用 DecoratedWnd 包装
{"MyVis", typeof(DecoratedWnd<ThirdParty.SomeWindow>) },
```

要求：被包装的窗口构造函数需接受一个 `Window` 参数作为 parent。

### 8.3 方案三：原生渲染集成（C++/CLI 桥接）

适用于：需要 OpenGL/VTK/DirectX 等原生渲染的高性能可视化。

**步骤：**

1. 创建 C++/CLI 项目（输出为 DLL），引用原生渲染库：

```cpp
// MyRender4wpf\my_render_clr.h
public ref class MyRender : INotifyPropertyChanged
{
public:
    MyRender(IntPtr parentHandle);
    void Initialize();
    void Render();
    void Resize(int width, int height);

    // 暴露属性供 WPF 绑定
    property double Zoom { double get(); void set(double); }
};
```

2. 创建 WinForms UserControl 作为宿主：

```csharp
// Helper\Visual\MyRenderView.cs
public class MyRenderView : System.Windows.Forms.UserControl
{
    public MyRender Renderer { get; private set; }

    protected override void OnHandleCreated(EventArgs e)
    {
        base.OnHandleCreated(e);
        Renderer = new MyRender(this.Handle);
        Renderer.Initialize();
    }
}
```

3. 在 WPF 窗口中通过 WindowsFormsHost 嵌入：

```xml
<!-- MyVisualizationWnd.xaml -->
<Grid x:Name="renderPanel">
    <WindowsFormsHost x:Name="formsHost" />
</Grid>
```

```csharp
// MyVisualizationWnd.xaml.cs
private void Window_Loaded(object sender, RoutedEventArgs e)
{
    var view = new MyRenderView();
    formsHost.Child = view;
}
```

4. 编译输出放到 `build\[Config]\bin\plugins\` 目录。

### 8.4 方案四：独立线程窗口

适用于：渲染负载重、需要独立消息循环的窗口。

```csharp
// 注册时使用 ThreadWnd 包装
{"MyVis", typeof(ThreadWnd<MyVisualizationWnd>) },
```

窗口将在独立 STA 线程中运行，拥有自己的 Dispatcher。

## 9. 集成注意事项

### 9.1 必须遵守

| 项目 | 要求 |
|------|------|
| 接口实现 | 必须实现 `ILocalWnd` 或使用 `DecoratedWnd<T>` 包装 |
| 构造函数 | 必须接受 `Window parent` 参数（反射实例化需要） |
| 线程模型 | WPF 为 STA，跨线程操作必须通过 Dispatcher |
| 目标框架 | .NET Framework 4.7.2, x64 |
| 生命周期 | 窗口关闭时触发 Closed 事件，主窗口据此清理引用 |

### 9.2 建议遵守

| 项目 | 建议 |
|------|------|
| 主题一致性 | 引用 `Colors.xaml` 中定义的颜色，使用 FontAwesome 图标 |
| MVVM 模式 | 使用数据绑定和 INotifyPropertyChanged，避免代码直接操作 UI |
| 资源管理 | 原生资源在窗口 Closed 时释放，避免内存泄漏 |
| 错误处理 | 原生调用包裹 try-catch，防止非托管异常崩溃整个应用 |
| 数据交互 | 通过模块系统的 Port 端口传递数据，不要直接跨窗口引用 |
| 本地化 | 字符串放入资源字典（参考 String.CN.xaml），不要硬编码 |

### 9.3 项目配置清单

新建插件项目时的 `.csproj` 关键配置：

```xml
<PropertyGroup>
    <TargetFramework>net472</TargetFramework>
    <PlatformTarget>x64</PlatformTarget>
    <OutputPath>..\..\build\$(Configuration)\bin\</OutputPath>
</PropertyGroup>

<ItemGroup>
    <!-- 引用核心库 -->
    <ProjectReference Include="..\Helper\Helper.csproj" />
</ItemGroup>
```

C++/CLI 原生插件的 CMake 配置要点：

```cmake
set(CMAKE_RUNTIME_OUTPUT_DIRECTORY "${CMAKE_SOURCE_DIR}/build/${CMAKE_BUILD_TYPE}/bin/plugins")
set_target_properties(my_render4wpf PROPERTIES
    COMMON_LANGUAGE_RUNTIME ""
    DOTNET_TARGET_FRAMEWORK "net472"
)
target_link_libraries(my_render4wpf PRIVATE
    ${VTK_LIBRARIES}  # 或其他原生库
)
```
