力感 CSV 字段说明（仅替换 ft_1/fn_1 来源）
====================================================================

文件对应：
- 主程序：main.cpp
- 日志结构：control_types.cpp / ForceLogState
- CSV 列顺序：
  timestamp,ft_1_value,fn_1_value,fn_2_value,ft_2_value,mode_code,reverse_code,push_pull_code,rot_sign_code,axis1_pos_rel

--------------------------------------------------------------------
1) 数据来源规则
--------------------------------------------------------------------
- 当 force_sample_source = ADS：
  ft_1/fn_1/fn_2/ft_2 均来自 ADS（PLC 变量）。

- 当 force_sample_source = TCP_DAQ：
  ft_1/fn_1 由 TCP 采集卡第0/1通道替换（0->ft_1，1->fn_1，按 INT16/1000.0 输出）；
  fn_2/ft_2 与其余字段继续来自 ADS 原有链路。

说明：
- TCP 的第2通道及之后在当前日志链路中不使用。
- 若该周期 TCP 无有效帧，则跳过该周期 CSV 行写入，并限频告警（1秒一次）。

--------------------------------------------------------------------
2) mode_code（当前模式编码）
--------------------------------------------------------------------
- 0：导管模式（guidewire_mode == None）
- 1：导丝模式（guidewire_mode != None）

--------------------------------------------------------------------
3) reverse_code（当前模式下的正反向状态）
--------------------------------------------------------------------
- 0：正向（未按下反向键）
- 1：反向（按下反向键）

--------------------------------------------------------------------
4) push_pull_code（推拉方向编码）
--------------------------------------------------------------------
- +1：活动模式下线性增量为正
- -1：活动模式下线性增量为负
-  0：线性增量为 0（静止或死区内）

--------------------------------------------------------------------
5) rot_sign_code（旋转符号方向编码）
--------------------------------------------------------------------
- +1：活动模式下旋转增量 > 死区
- -1：活动模式下旋转增量 < -死区
-  0：旋转增量在死区内
