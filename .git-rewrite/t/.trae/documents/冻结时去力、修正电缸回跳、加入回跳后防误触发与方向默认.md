## 我对现象的解释（先把“量怎么变”讲清楚）

### 1) axis1\_push\_sign = -1 时，手柄推拉→轴1/3 的量变化关系

以下都以你给的现场观测为准：\**手柄推动时 J1(=fJoints2\[0]1000 显示)增大，但电机位置(轴1绝对 ActPos)减小*。

* **手柄侧（上位机）**

  * `J1 = handle.fJoints2[0]`（单位 mm/1000，程序里乘 500 做 mm 映射）

  * 参考点：`handle_ref`（同步时取一次均值，之后作为零点）

  * 映射量：

    * `hand_delta_mm = (J1 - handle_ref) * k_handle_to_mm * axis1_push_sign`

    * 当 `axis1_push_sign = -1` 且“推动→J1增大”，则 `hand_delta_mm < 0`

  * 轴1目标（绝对）：

    * `axis1_base_abs = axis1_base_rel + plc_init_pos[0]`（同步时读回的当前绝对位置）

    * `cmd_abs = axis1_base_abs + hand_delta_mm`

    * 因为 `hand_delta_mm < 0`，所以 **推动→cmd\_abs 变小**。

* **PLC侧（handle.TcPOU）**

  * PLC每周期算：`G.Act_pos[1] = Axis1.AbsActPos - G.init_pos[1]`（相对）

  * 上位机写入：`G.refer[1] = cmd_abs - G.init_pos[1]`（相对参考）

  * PLC平滑：`G.ref_slow` 以 `G.v_limit[1]`（或 fast\_return 时 1000）跟随 `G.refer[1]`

  * PLC输出：`G.act_h[1]`（平均）再喂给 `MC_ExtSetPointGenFeed(Position := G.act_h[1] + G.init_pos[1])`

  * 所以：**cmd\_abs 变小 → G.refer\[1] 变小 → 下发绝对 Position 变小 → 电机绝对位置变小**。

* **“前进/后退”的概念（以 rightlimit 为参照）**

  * rightlimit 是“右限位的绝对坐标”。

  * 你现在定义的两个端点（都是绝对坐标）：

    * `StartAbs = rightlimit - 56`

    * `EndAbs   = rightlimit - 36`

  * 若轴1绝对位置 **变小**，则 `rightlimit - axis1_abs` 变大，表示“离右限位更远”。

  * 因此在你当前现场语义里：

    * **推动（J1增大）→轴1绝对变小 → 离右限位更远**：这就是你说的“前进/向前”方向。

    * **回拉（J1减小）→轴1绝对变大 → 离右限位更近**：对应“后退/向后”。

  * 轴3镜像：正常段 `pos[2] = axis3_base_rel + (pos[0]-axis1_base_rel)`，所以它会随轴1做同样“绝对变小/变大”的趋势；在轴1快回跳阶段暂停更新。

### 2) b7 按下后手柄仍有力的原因（你触感到“推着手柄”）

现在 b7 的冻结只做了“停止写 refer/停止控制运动”，但**力反馈那段仍可能在某些分支里留下非零力（尤其是滤波记忆/或某次没走到清零分支）**，导致你感觉还在推。

目标行为应该是：**b7按下后，手柄输出力应立刻归零，并清空滤波记忆**。

### 3) 快回跳时电缸没按预期动作的可能原因

* **最常见**：电缸动作时间比当前等待（`axis1_open_delay_ms=120ms`）更长，导致轴1已经开始“最大速度回跳”，但夹持状态没真正切换到位。

* **其次**：在快回跳期间虽然程序写了电缸值，但你现场电缸驱动/阀响应慢，仍然需要更长的“开/夹等待”。

## 你提的“轻轻反方向推动就误触发又快回跳”——根因与解决

根因：轴1刚回跳到端点后，**轴1实际仍处在端点容差范围内**，此时你手柄轻微反向，`hand_delta_mm` 符号一变，触发条件（“在端点附近 + hand\_delta 符号”）就再次成立。

解决方式建议两层（你提的思路属于第1层）：

1. **方向锁存 + 方向死区（推荐）**

   * 当一次回跳完成后，锁存“本次蠕动的有效方向”（比如这是“推手柄蠕动”还是“拉手柄蠕动”）。

   * 回到端点后，只有当手柄在“同方向”上的控制量超过阈值（例如 0.5mm 等效）才允许进入下一次触发；反方向小于阈值的一律当作误差/回弹。
2. **触发条件改成“指令到端点”而不是“实际在端点”**

   * 目前用 `abs(axis1_abs - endpoint) < tol` 来触发。

   * 可以改为：`abs(cmd_abs - endpoint) < small_tol` 且 `abs(hand_delta_mm) > deadband`，这样手柄轻微反向时 cmd\_abs 已经离开端点，不会误触发。
